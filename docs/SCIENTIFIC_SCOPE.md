# Scientific Scope

## What this repository validates

- Layer A validates simulator-internal mechanism claims under finite context.
- Layer B validates whether some controller-law contrasts remain visible on small bundled mechanistic probe tasks.
- The repository validates reproducible logging, analysis, and fixed-backbone comparison workflows.

## What this repository does not validate

- It does not establish world-exact truth about real agents or production systems.
- It does not establish broad benchmark validity from the bundled micro-task slice.
- It does not prove the theory universally.
- It does not show that any one controller dominates across all settings.

## Why Layer A matters

Layer A is where the theory is tested most directly. Its exactness is simulator-internal exactness:

- the adequate-family set is known by construction
- recoverability is known by construction
- failure-channel attribution is known by construction

That makes Layer A the main source for mechanism-level claims.

## Why Layer B is currently limited

Layer B uses task outcomes and deterministic family proxies, not latent world truth.

Its current bundled tasks are:

- public-safe
- lightweight
- mechanistic
- intentionally small

This makes Layer B useful for probe-style controller comparisons, but not for broad external validation.

## What stronger future validation would require

- a larger frozen task slice
- stronger task diversity
- cleaner local structured-output reliability
- broader backbone replication
- careful benchmark or external task governance

## Safe reading rule

Use Layer A for exact mechanism claims.
Use Layer B for proxy-limited directional evidence only.
